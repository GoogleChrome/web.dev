web.dev content checklist

Jun 26, 2019
Michael Friesenhahn
  • GitHub

This is a checklist for web.dev content reviews. It includes critical high-level issues and lower-level issues that commonly appear in drafts.

Basics #

  1. Is the piece providing value for one or more of the web.dev personas?
  2. Do the title, subtitle, and description convey the value proposition?
  3. Is the piece 1,000 words or lessâ€”or is there a good reason for it to be longer?
  4. Are all statements and advice accurate and aligned with best practices?

Style and voice #

  1. Is the voice appropriate for the web.dev audience?
  2. Can any language be removed without changing the meaning or voice?
  3. Is the language as simple as possible? Is there any unnecessary jargon?

Organization #

  1. Is the structure of the piece clear? Do headings convey the structure?
  2. Does each paragraph focus on one main idea?

Instruction #

  1. If the piece expects readers to have prior knowledge, is that made clear up front?
  2. Does the piece explain why the process being taught matters?
  3. Are examples provided to support all key claims and ideas?
  4. Are instructions in second person (you/your), not first person (we/our)?
  5. Are common instructions provided using the web.dev Instruction components?

Media #

  1. Are images and videos used to clarify ideas that would be difficult to understand from text alone?
  2. Are any images or videos included that don't directly relate to ideas in the text?
  3. Are image captions correctly formatted?
  4. Do all images have alt text?

Code #

  1. Are code blocks and sample apps as simple as possible while still conveying the core concept?
  2. Is a brief description of a sample app's functionality provided before the app itself?
  3. Is code highlighting used to indicate lines that have been added or changed?
  4. Is all sample code accessible?

Links #

  1. Do all links work?
  2. Are all links to web.dev pages and assets relative?

Mechanics #

  1. Is the text free of spelling and capitalization errors? (Check the word list.)
  2. Are dashes and hyphens used correctly?
  3. Are titles and headings in sentence case with no terminal period?
  4. Are keyboard key commands correctly formatted?
