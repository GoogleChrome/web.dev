Quick start guide

Jun 26, 2019
Michael Friesenhahn
  • GitHub

Content creation for web.dev has three phases: planning, writing, and publishing.

Planning #

  1. Start a content proposal issue on GitHub using this issue template. The checklist in your issue will guide you through the rest of the process.
  2. Create a copy of this document template to briefly write up your content idea. Put the link to your doc in your GitHub issue.
  3. The web.dev team will take a look to see if the idea fits with the goals of the site. If the idea is approved, it gets slated for a publication date!

Caution: If the piece you'd like to publish is time sensitive, make sure to submit the issue at least two weeks before the target publication date so there's enough time to move it through the writing process.

Writing #

  1. Write a draft in the Google doc you used to propose your piece. (Before you start, read our style guidelines.)
  2. When you're ready, request a content review in the GitHub issue.
  3. The tech writer will review using the web.dev content checklist and work with you to polish the piece.

Publishing #

  1. Once you and the tech writer think the piece is good to go, submit a pull request on GitHub with your content authored in Markdown. (Check out the "web.dev markup" section for details.)
  2. The web.dev team will give your Markdown a once over. If everything looks good, your pull request is merged, and your piece is published! ðŸŽ‰
